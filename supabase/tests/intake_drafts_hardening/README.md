# Intake Draft hardening — security validation harness

Self-contained local-PostgreSQL harness that proves migration
`20260717090000_capability2_slice2a_intake_drafts_hardening.sql` against the
schema created by `20260716120000_capability2_slice2a_intake_drafts.sql`.

It provides its **own** minimal base stubs (Supabase `auth.uid()/auth.role()`,
roles `authenticated`/`anon`/`service_role`, `app.is_org_member/is_org_admin/
touch_updated_at`, and the base tables) so it can run on a throwaway cluster
without the full application schema. `30_tests.sql` is generated by
`gen_tests.py` (kept for regeneration).

## Run order (throwaway local Postgres)

```
psql -f 00_base.sql                                                   # stubs + base tables
psql -f ../../migrations/20260716120000_capability2_slice2a_intake_drafts.sql
psql -f 10_grants.sql                                                 # table DML grants
psql -f ../../migrations/20260717090000_capability2_slice2a_intake_drafts_hardening.sql
psql -f 20_seed.sql                                                   # one draft per lifecycle state
psql -f 30_tests.sql                                                  # 67 checks -> public._results
psql -c "select test_no, passed, detail from public._results order by test_no;"
```

Both reconciliation paths pass **67/67**:
- clean historical → hardening, and
- historical + out-of-band InitPlan ALTER (live Development state) → hardening.
